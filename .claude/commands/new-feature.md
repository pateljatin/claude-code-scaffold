Use this command to plan before implementing. Do not write code until the plan is approved.

Steps to follow:

1. Clarify the requirement: what exactly needs to change, and why?
2. Identify all files that will be created or modified.
3. Outline the implementation approach: data flow, component design, API shape, DB changes.
4. Note risks, edge cases, and what must be tested.
5. Estimate complexity and propose model+effort.

Output a numbered plan. Wait for explicit user approval before writing any code.
