---
description: Commit staged changes, push, and open a pull request
---

1. Run the test suite. Stop and report if anything fails.
2. Review staged changes: confirm no unintended files are included.
3. Write a commit message: imperative mood, under 72 chars on the first line, no em-dashes.
4. Commit and push to the current branch.
5. Open a PR with `gh pr create`. Title: imperative, under 72 chars. Body: what changed and why, not how.
6. Report the PR URL.
