# Spec Rules

## Writing Style

- No em-dashes. Use commas, periods, or semicolons instead.
- No "AI-powered," "leverage," "seamless," "robust," "revolutionary," "transformative," or "ecosystem."
- Oxford comma. US English.
- Write like a builder, not a marketer.

## Code

- No comments unless the WHY is non-obvious.
- No error handling for impossible cases. Trust internal invariants.
- No half-implementations or placeholder stubs left in the tree.
- YAGNI. Three similar lines before abstracting.
- Validate only at system boundaries: user input, external APIs, file system reads.
