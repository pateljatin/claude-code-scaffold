Why it matters

What you actually gain

Each benefit maps directly to a specific file. These are not features — they are problems that bite without them.

📈

Context survives compaction

When Claude compacts a long conversation, the PostCompact hook reinjects your project facts from .claude/context.md. Claude never forgets what it's working on.

💵

Right model, right cost

A mandatory model+effort rule means Haiku handles grep, Sonnet handles features, and Opus is reserved for architecture. No silent Opus runs on trivial edits.

🚫

Standards enforced at write time

The PreToolUse hook blocks banned phrases ("AI-powered," "leverage," "seamless") and em-dashes before they hit a file — not after a manual review catches them.

🛡

Dangerous operations blocked

The PermissionRequest hook auto-allows Read, Glob, Grep and blocks rm -rf, DROP TABLE, git push --force before they can run.

⚙

Repeatable workflows

Slash commands make commit, PR, post-merge, and feature planning identical every time — same steps, same checks, same output, regardless of who runs them.

🔐

PR assistant that can't be hijacked

Comment content is written via env var, never shell-interpolated. Author association and fork checks block unauthorized triggers before Claude ever runs.

Architecture

Hook lifecycle

Five hooks, five scripts. Each is PowerShell with bash equivalents in comments.

.claude/hooks/
UserPromptSubmit PreToolUse | | v v session-start-habits.ps1 banned-word-check.ps1 fires once per session checks every write for banned phrases prints habits reminder exit 2 = hard block PostCompact Stop | | v v post-compact-reinject.ps1 stop-baseline-check.ps1 reads .claude/context.md scans changed files for TODO/FIXME outputs it back into context warns before Claude signs off PermissionRequest | v permission-request-router.ps1 auto-allows: Read, Glob, Grep, LS blocks: rm -rf DROP TABLE git push --force

Slash commands

Built-in workflows

Each command is a markdown file in .claude/commands/. Add your own for project-specific workflows.

CommandWhat it does
/scaffold Interactive setup. Five questions inside Claude Code. Writes every configured file, offers to create the GitHub repo.
/new-feature Plan before coding. Identifies files, approach, risks, model+effort. No code until approved. planning only
/post-merge Post-merge checklist. Pull, delete branch, run tests, check deps, update docs. Fails loud if tests are red. planning only
/commit-push-pr Ship a change. Runs tests, stages, commits with a well-formed message, pushes, opens PR with gh pr create.
/go Pick up the next task. Checks branch, open PRs, active goal, proposes model+effort, waits for confirmation.

Sub-agents

Specialists on demand

Defined in .claude/agents/. Spawned with scoped context and minimal tool permissions.

AgentPurpose
verify-app Run and confirm. Starts the app, navigates to the relevant flow, checks for console errors and regressions, returns pass/fail with evidence.
security-reviewer Review before merge. Checks diffs for injection, hardcoded secrets, missing auth, path traversal, and known vulnerable dependencies.
test-writer Fill coverage gaps. Reads the module, matches existing test conventions, writes tests for happy path and edge cases, runs them, reports results.

Get started

Install

Use this repo as a GitHub template, or apply the scaffold to an existing project with the setup scripts.

1
Use this template on GitHub Click "Use this template" on the repo page. Clone your new repo locally.
2
Open Claude Code in the directory Run /scaffold in the Claude Code prompt.
3
Answer five questions Project name, your name, description, stack, GitHub username. Claude writes every configured file.
4
Optionally create the GitHub repo Claude offers to run gh repo create at the end.

Or: apply to an existing project

git clone https://github.com/pateljatin/claude-code-scaffold
cd claude-code-scaffold
.\setup.ps1 -Target C:\path\to\your-project
git clone https://github.com/pateljatin/claude-code-scaffold
cd claude-code-scaffold
./setup.sh /path/to/your-project
